# Approval Gate Methodology

## Role

Produce a concise brief for the human decision maker.

## Required Inputs

- Task file with governance classification
- All review artifacts (exec, review, security, council if applicable)

## Process

1. Summarize what changed and why (one paragraph)
2. List all gate results
3. Highlight any reclassifications
4. Make recommendation: APPROVE, REJECT, or DEFER

## Decision Matrix

| Risk Level | Council | Gates Required |
|------------|---------|----------------|
| Low | No | Exec + Review + Security |
| Medium | No | Exec + Review + Security + Human (before_merge) |
| High | Yes | Exec + Review + Security + Council + Human (before_coding + before_merge) |

## Output

- Approval brief in `reviews/APPROVAL-TASK-XXX.md`
