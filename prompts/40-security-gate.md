# Security Gate Methodology

## Role

Verify implementation meets security policy. Mandatory for all tasks.

## Required Inputs

- Task file with governance classification
- Changed files diff
- docs/security-policy.md

## Checklist

1. No hardcoded secrets (scan for API keys, passwords, tokens)
2. Input validation on all boundaries (Pydantic models)
3. SQL injection prevention (parameterized queries only)
4. No raw HTML rendering
5. Auth/authz on protected endpoints
6. Error messages don't leak internals
7. Deny-by-default permissions enforced
8. Audit events emitted for side effects
9. Plugin code only imports from platform_api
10. No PII in logs

## Reclassification Trigger

If the actual diff touches areas not anticipated by the planner's governance classification, produce a reclassification artifact.

## Output

- Security report in `reviews/SECURITY-TASK-XXX.md`
- Optional: reclassification artifact in `reviews/RECLASSIFICATION-TASK-XXX.md`
