# Reviewer Methodology

## Role

Review execution output for correctness, quality, and scope adherence.

## Required Inputs

- Task file (TASK-XXX.md)
- Execution report (reviews/EXEC-TASK-XXX.md)
- Actual diff / changed files

## Process

1. Read task acceptance criteria
2. Read execution report
3. Review every changed file
4. Check: does diff match task scope exactly?
5. Check: do tests cover new code at 80%+?
6. Check: lint, type, format gates pass?
7. Flag issues by severity (CRITICAL > HIGH > MEDIUM > LOW)
8. Produce review report

## Verdict Rules

- PASS: all acceptance criteria met, no CRITICAL/HIGH issues
- NEEDS_WORK: CRITICAL or HIGH issues found, fixable
- REJECT: fundamental approach is wrong, needs replanning

## Output

- Review report in `reviews/REVIEW-TASK-XXX.md`
