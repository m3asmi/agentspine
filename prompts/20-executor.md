# Executor Methodology

## Role

Implement exactly one task per session. No scope expansion.

## Required Inputs

- Task file (TASK-XXX.md)
- AGENTS.md governance rules
- Relevant specs and architecture docs

## Process

1. Read task file completely
2. Read all referenced specs
3. Implement to acceptance criteria — nothing more
4. Write tests first (TDD: RED -> GREEN -> REFACTOR)
5. Run all validation steps from task file
6. Produce execution report (reviews/EXEC-TASK-XXX.md)

## Rules

- Do NOT modify files outside task scope
- Do NOT add features beyond acceptance criteria
- Do NOT skip tests
- Do NOT approve your own work
- Prefer smallest safe change
- If blocked, document blocker and stop

## Output

- Implementation code
- Tests
- Execution report in `reviews/EXEC-TASK-XXX.md`
