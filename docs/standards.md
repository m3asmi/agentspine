# Coding Standards

## Language & Runtime

- **Python**: 3.12.13
- **Package manager**: `uv`
- **Style**: PEP 8 with modifications below

## Formatting & Linting

- **Formatter/Linter**: Ruff (`ruff check .`, `ruff format .`)
- **Line length**: 100 characters
- **Target version**: py312
- **Type checker**: mypy (strict mode)

## Type Hints

- All public functions must have complete type annotations
- Use `from __future__ import annotations` for forward references
- Strict mypy: `disallow_untyped_defs = true`

## Naming Conventions

| Element | Style | Example |
|---------|-------|---------|
| Module | snake_case | `manifest_loader.py` |
| Class | PascalCase | `PluginRegistry` |
| Function/Method | snake_case | `load_manifest()` |
| Constant | UPPER_SNAKE | `MAX_RETRY_COUNT` |
| Private | leading underscore | `_internal_state` |
| Protocol/ABC | suffix with Base/Protocol | `ExtensionBase` |

## Error Handling

- Use typed exceptions inheriting from a project base exception
- Never silently swallow errors
- Log detailed context on the server side
- Return user-friendly messages in API responses
- Use `raise ... from` for exception chaining

## Input Validation

- Validate all external input with Pydantic models
- Fail fast with clear error messages
- Schema-first IO for all API boundaries

## File Organization

- Target: 200-400 lines per file, 800 max
- One class per file for major abstractions
- Organize by feature/domain, not by type
- Keep `__init__.py` files minimal (re-exports only)

## Immutability

- Prefer `frozen=True` on Pydantic models where possible
- Return new objects instead of mutating existing ones
- Use `tuple` over `list` for immutable sequences

## Imports

- Standard library first, then third-party, then local
- Use absolute imports from `agent_framework.*`
- No wildcard imports (`from x import *`)

## Documentation

- Docstrings on all public classes and functions
- Use Google-style docstrings
- Keep inline comments for non-obvious logic only
