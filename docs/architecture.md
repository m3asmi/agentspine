# Architecture Constraints

## Mission Statement

Build a local-first, plugin-ready AI agent framework with strict runtime boundaries, auditable side effects, and provider-agnostic LLM routing.

## Key Architecture Decisions

- Python 3.12 with `uv` as package manager.
- FastAPI for API entrypoints.
- LangGraph for orchestration state graphs.
- PostgreSQL + pgvector for durable state and vector retrieval.
- Langfuse for observability and traceability.
- Ollama for local model execution.
- OpenAI-compatible gateway abstraction for remote model access.
- Plugin-first extension model through typed public interfaces and registries.

## Hard Constraints

- Do not bypass plugin system contribution points.
- Do not hardcode provider-specific model calls in orchestration nodes.
- All side effects must emit audit events.
- Deny-by-default permissions for capabilities and executors.
- Public extension contracts must remain stable and versioned.
- Phase execution order is mandatory unless marked parallel-safe.

## Repository Map

- `src/agent_framework/main.py` — FastAPI app entrypoint.
- `src/agent_framework/platform_api/` — public extension contracts.
- `src/agent_framework/registries/` — extension and capability registries.
- `src/agent_framework/lifecycle/` — startup/shutdown and plugin lifecycle orchestration.
- `src/agent_framework/llm_gateway/` — provider-agnostic model routing.
- `src/agent_framework/policy/` — permissions and policy enforcement.
- `src/agent_framework/audit/` — audit event emission.
- `src/agent_framework/execution/` — execution dispatcher and runtime controls.
- `src/agent_framework/plugins/` — plugin loading and composition.
- `src/agent_framework/orchestration/` — LangGraph orchestration nodes/graphs.
- `src/agent_framework/knowledge/` and `src/agent_framework/memory/` — retrieval/memory primitives.
- `tests/` — automated tests (currently includes health checks and phase tests).
- `docs/` — architecture/spec/governance documentation.
- `tasks/` — scoped execution tasks and dependency ordering.
- `reviews/` — execution/review/security/approval artifacts.

## Package and Dependency Policy

- Runtime dependencies are pinned via `uv.lock`.
- New dependencies require explicit justification in task acceptance criteria.
- Prefer standard library and existing stack before adding libraries.
- Keep integration adapters behind interfaces to avoid vendor lock-in.

## Testing Gates

- `uv run ruff check .`
- `uv run ruff format --check .`
- `uv run mypy .`
- `uv run pytest`
- `uv run pytest --cov` (minimum 80% coverage)
